# 人员信息接口文档

本文档记录当前服务已经实现的人员/认证相关接口，方便前端或接口调试工具调用。

## 基础信息

- 服务地址：`http://127.0.0.1:8080`
- 数据格式：`application/json`
- 鉴权方式：`Authorization: Bearer <token>`

需要登录的接口必须在请求头中携带登录或注册接口返回的 `token`。

```http
Authorization: Bearer <token>
```

## 通用响应

成功响应会根据接口返回 `user`、`users` 或 `token`。

错误响应统一返回：

```json
{
  "error": "错误信息"
}
```

当前常见错误信息：

| HTTP 状态码 | error |
| --- | --- |
| 400 | 参数校验失败 |
| 401 | 未登录或登录已失效 |
| 401 | 账号或密码错误 |
| 403 | 用户已禁用 |
| 409 | 用户名已存在 |
| 500 | 服务器内部错误 |

## 用户对象

接口返回的用户对象不会包含密码字段。

```json
{
  "id": 1,
  "external_id": "",
  "username": "alice",
  "nickname": "Alice",
  "avatar_url": "https://example.com/avatar.png",
  "auth_source": "local",
  "status": 1
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | number | 用户内部 ID |
| external_id | string | 外部系统用户 ID，本地注册用户通常为空 |
| username | string | 登录账号 |
| nickname | string | 展示昵称 |
| avatar_url | string | 头像地址，可为空 |
| auth_source | string | 用户来源，当前本地注册为 `local` |
| status | number | 用户状态，`1` 正常，`0` 禁用 |

## 注册

创建本地用户，注册成功后直接返回用户信息和登录 token。

- 方法：`POST`
- 路径：`/api/auth/register`
- 是否需要登录：否

### 请求参数

```json
{
  "username": "alice",
  "password": "123456",
  "nickname": "Alice",
  "avatar_url": "https://example.com/avatar.png"
}
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| username | string | 是 | 登录账号，必须唯一 |
| password | string | 是 | 登录密码 |
| nickname | string | 否 | 展示昵称，不传时默认使用 username |
| avatar_url | string | 否 | 头像地址 |

### 成功响应

HTTP 状态码：`201`

```json
{
  "token": "<jwt-token>",
  "user": {
    "id": 1,
    "username": "alice",
    "nickname": "Alice",
    "avatar_url": "https://example.com/avatar.png",
    "auth_source": "local",
    "status": 1
  }
}
```

### 失败响应

| HTTP 状态码 | error | 场景 |
| --- | --- | --- |
| 400 | 参数校验失败 | username 或 password 为空，或 JSON 格式不符合要求 |
| 409 | 用户名已存在 | username 已被注册 |
| 500 | 服务器内部错误 | 数据库或 token 签发失败 |

### curl 示例

```bash
curl -X POST http://127.0.0.1:8080/api/auth/register \
  -H 'Content-Type: application/json' \
  -d '{
    "username": "alice",
    "password": "123456",
    "nickname": "Alice",
    "avatar_url": "https://example.com/avatar.png"
  }'
```

## 登录

使用本地账号密码登录，成功后返回用户信息和新的 token。

- 方法：`POST`
- 路径：`/api/auth/login`
- 是否需要登录：否

### 请求参数

```json
{
  "username": "alice",
  "password": "123456"
}
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| username | string | 是 | 登录账号 |
| password | string | 是 | 登录密码 |

### 成功响应

HTTP 状态码：`200`

```json
{
  "token": "<jwt-token>",
  "user": {
    "id": 1,
    "username": "alice",
    "nickname": "Alice",
    "avatar_url": "https://example.com/avatar.png",
    "auth_source": "local",
    "status": 1
  }
}
```

### 失败响应

| HTTP 状态码 | error | 场景 |
| --- | --- | --- |
| 400 | 参数校验失败 | username 或 password 为空，或 JSON 格式不符合要求 |
| 401 | 账号或密码错误 | 用户不存在或密码错误 |
| 403 | 用户已禁用 | 用户状态为禁用 |
| 500 | 服务器内部错误 | token 签发失败 |

### curl 示例

```bash
curl -X POST http://127.0.0.1:8080/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{
    "username": "alice",
    "password": "123456"
  }'
```

## 获取当前用户

根据请求头中的 token 返回当前登录用户信息，也可用于前端检查 token 是否仍然有效。

- 方法：`GET`
- 路径：`/api/me`
- 是否需要登录：是

### 请求头

```http
Authorization: Bearer <token>
```

### 成功响应

HTTP 状态码：`200`

```json
{
  "user": {
    "id": 1,
    "username": "alice",
    "nickname": "Alice",
    "avatar_url": "https://example.com/avatar.png",
    "auth_source": "local",
    "status": 1
  }
}
```

### 失败响应

| HTTP 状态码 | error | 场景 |
| --- | --- | --- |
| 401 | 未登录或登录已失效 | 未携带 token、token 无效、token 过期、用户不存在或用户被禁用 |

### curl 示例

```bash
curl http://127.0.0.1:8080/api/me \
  -H 'Authorization: Bearer <token>'
```

## 获取用户列表

返回数据库中的所有用户信息。当前接口用于管理和调试，后续可扩展分页、搜索和角色权限。

- 方法：`GET`
- 路径：`/admin/users`
- 是否需要登录：是

### 请求头

```http
Authorization: Bearer <token>
```

### 成功响应

HTTP 状态码：`200`

```json
{
  "users": [
    {
      "id": 2,
      "username": "bob",
      "nickname": "Bob",
      "avatar_url": "",
      "auth_source": "local",
      "status": 1
    },
    {
      "id": 1,
      "username": "alice",
      "nickname": "Alice",
      "avatar_url": "https://example.com/avatar.png",
      "auth_source": "local",
      "status": 1
    }
  ]
}
```

### 失败响应

| HTTP 状态码 | error | 场景 |
| --- | --- | --- |
| 401 | 未登录或登录已失效 | 未携带 token、token 无效、token 过期、用户不存在或用户被禁用 |
| 500 | 服务器内部错误 | 查询用户列表失败 |

### curl 示例

```bash
curl http://127.0.0.1:8080/admin/users \
  -H 'Authorization: Bearer <token>'
```

## 调用顺序建议

1. 调用 `POST /api/auth/register` 注册用户并保存返回的 `token`。
2. 或调用 `POST /api/auth/login` 登录并保存返回的 `token`。
3. 后续访问 `GET /api/me`、`GET /admin/users` 时，在请求头中携带 `Authorization: Bearer <token>`。
