# 人员信息接口文档

本文档记录当前服务已经实现的人员/认证相关接口，方便前端或接口调试工具调用。

## 基础信息

- 服务地址：`http://127.0.0.1:8080`
- 数据格式：`application/json`
- 配置文件：`conf/app.ini`
- 鉴权方式：`Authorization: Bearer <token>`

## 统一响应结构

所有接口统一返回：

```json
{
  "code": 200,
  "data": {},
  "message": "接口信息"
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | number | 状态码，成功为 `200`，失败时为对应错误状态码 |
| data | any | 数据源；失败时为 `null` |
| message | string | 接口提示信息；接口报错时放错误信息 |

错误响应示例：

```json
{
  "code": 400,
  "data": null,
  "message": "参数校验失败"
}
```

## 用户对象

接口返回的用户对象不会包含密码字段。

```json
{
  "id": 1,
  "external_id": "",
  "username": "alice",
  "nickname": "Alice",
  "avatar_url": "https://example.com/avatar.png",
  "auth_source": "local",
  "status": 1
}
```

## 注册

- 方法：`POST`
- 路径：`/api/auth/register`
- 是否需要登录：否

### 请求参数

```json
{
  "username": "alice",
  "password": "123456",
  "nickname": "Alice",
  "avatar_url": "https://example.com/avatar.png"
}
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| username | string | 是 | 登录账号，必须唯一 |
| password | string | 是 | 登录密码 |
| nickname | string | 否 | 展示昵称，不传时默认使用 username |
| avatar_url | string | 否 | 头像地址 |

### 成功响应

```json
{
  "code": 200,
  "data": {
    "token": "<jwt-token>",
    "user": {
      "id": 1,
      "username": "alice",
      "nickname": "Alice",
      "avatar_url": "https://example.com/avatar.png",
      "auth_source": "local",
      "status": 1
    }
  },
  "message": "注册成功"
}
```

### 失败响应

| code | message | 场景 |
| --- | --- | --- |
| 400 | 参数校验失败 | username 或 password 为空，或 JSON 格式不符合要求 |
| 409 | 用户名已存在 | username 已被注册 |
| 500 | 服务器内部错误 | 数据库或 token 签发失败 |

## 登录

- 方法：`POST`
- 路径：`/api/auth/login`
- 是否需要登录：否

### 请求参数

```json
{
  "username": "alice",
  "password": "123456"
}
```

### 成功响应

```json
{
  "code": 200,
  "data": {
    "token": "<jwt-token>",
    "user": {
      "id": 1,
      "username": "alice",
      "nickname": "Alice",
      "avatar_url": "https://example.com/avatar.png",
      "auth_source": "local",
      "status": 1
    }
  },
  "message": "登录成功"
}
```

### 失败响应

| code | message | 场景 |
| --- | --- | --- |
| 400 | 参数校验失败 | username 或 password 为空，或 JSON 格式不符合要求 |
| 401 | 账号或密码错误 | 用户不存在或密码错误 |
| 403 | 用户已禁用 | 用户状态为禁用 |
| 500 | 服务器内部错误 | token 签发失败 |

## 获取当前用户

- 方法：`GET`
- 路径：`/api/me`
- 是否需要登录：是

### 请求头

```http
Authorization: Bearer <token>
```

### 成功响应

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "username": "alice",
    "nickname": "Alice",
    "avatar_url": "https://example.com/avatar.png",
    "auth_source": "local",
    "status": 1
  },
  "message": "获取当前用户成功"
}
```

### 失败响应

| code | message | 场景 |
| --- | --- | --- |
| 401 | 未登录或登录已失效 | 未携带 token、token 无效、token 过期、用户不存在或用户被禁用 |

## 获取用户列表

- 方法：`GET`
- 路径：`/admin/users`
- 是否需要登录：是

### 请求头

```http
Authorization: Bearer <token>
```

### 成功响应

```json
{
  "code": 200,
  "data": [
    {
      "id": 2,
      "username": "bob",
      "nickname": "Bob",
      "avatar_url": "",
      "auth_source": "local",
      "status": 1
    },
    {
      "id": 1,
      "username": "alice",
      "nickname": "Alice",
      "avatar_url": "https://example.com/avatar.png",
      "auth_source": "local",
      "status": 1
    }
  ],
  "message": "获取用户列表成功"
}
```

### 失败响应

| code | message | 场景 |
| --- | --- | --- |
| 401 | 未登录或登录已失效 | 未携带 token、token 无效、token 过期、用户不存在或用户被禁用 |
| 500 | 服务器内部错误 | 查询用户列表失败 |

## 调用顺序建议

1. 调用 `POST /api/auth/register` 注册用户并保存返回的 `data.token`。
2. 或调用 `POST /api/auth/login` 登录并保存返回的 `data.token`。
3. 后续访问 `GET /api/me`、`GET /admin/users` 时，在请求头中携带 `Authorization: Bearer <token>`。
