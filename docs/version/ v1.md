# 版本一：用户体系基础版

当前 v1 文档只描述代码中已经实现的用户相关能力。

## 已实现范围

- 本地用户注册
- 本地账号密码登录
- 注册或登录成功后签发 JWT
- 基于 `Authorization: Bearer <token>` 的接口鉴权
- 根据 token 获取当前用户信息
- 获取用户列表的管理/调试接口

## 本地启动

服务默认读取项目根目录下的 `conf/app.ini`，其中包含 MySQL、HTTP、JWT 配置。

启动命令：

```bash
go run .
```

使用 fresh 热重载：

```bash
fresh
```

默认本地服务地址：

```text
http://127.0.0.1:8080
```

## 数据库

当前代码只依赖 `users` 表。

建表文档：[数据库表创建-人员](../数据库/数据库表创建-人员.md)

当前 `users` 表支持：

- `username` 唯一账号
- `password` 本地登录密码
- `nickname` 展示昵称
- `avatar_url` 头像地址
- `external_id` 和 `auth_source`
- `status` 用户状态，`1` 表示正常，`0` 表示禁用

当前代码中 `password` 按明文保存和比较，接口响应通过 DTO 过滤，不返回密码字段。

## 接口约定

当前所有接口统一返回：

```json
{
  "code": 200,
  "data": {},
  "message": "操作成功"
}
```

失败时 `code` 与 HTTP 状态码保持一致，`data` 为 `null`。

### 注册

```text
POST /api/auth/register
```

请求体：

```json
{
  "username": "alice",
  "password": "123456",
  "nickname": "Alice",
  "avatar_url": "https://example.com/avatar.png"
}
```

字段说明：

- `username`：必填，本地登录账号
- `password`：必填，本地登录密码
- `nickname`：选填，不传时默认使用 `username`
- `avatar_url`：选填，头像地址

成功响应：

```json
{
  "code": 200,
  "data": {
    "user": {
      "id": 1,
      "username": "alice",
      "nickname": "Alice",
      "avatar_url": "https://example.com/avatar.png",
      "auth_source": "local",
      "status": 1
    },
    "token": "<jwt-token>"
  },
  "message": "注册成功"
}
```

常见错误：

- `400`：参数校验失败
- `409`：用户名已存在
- `500`：服务器内部错误

### 登录

```text
POST /api/auth/login
```

请求体：

```json
{
  "username": "alice",
  "password": "123456"
}
```

成功响应：

```json
{
  "code": 200,
  "data": {
    "user": {
      "id": 1,
      "username": "alice",
      "nickname": "Alice",
      "avatar_url": "https://example.com/avatar.png",
      "auth_source": "local",
      "status": 1
    },
    "token": "<jwt-token>"
  },
  "message": "登录成功"
}
```

常见错误：

- `400`：参数校验失败
- `401`：账号或密码错误
- `403`：用户已禁用
- `500`：服务器内部错误

```json
{
```

### 获取用户列表

```text
GET /admin/users
```

请求头：

```text
Authorization: Bearer <jwt-token>
```

说明：

当前接口用于管理和调试，返回数据库里的用户列表。接口已挂 JWT 鉴权，但还没有做管理员角色控制。

成功响应：

```json
{
  "code": 200,
  "data": [
    {
      "id": 2,
      "username": "bob",
      "nickname": "Bob",
      "avatar_url": "",
      "auth_source": "local",
      "status": 1
    },
    {
      "id": 1,
      "username": "alice",
      "nickname": "Alice",
      "avatar_url": "https://example.com/avatar.png",
      "auth_source": "local",
      "status": 1
    }
  ],
  "message": "获取用户列表成功"
}
```

常见错误：

- `401`：未登录或登录已失效
- `500`：服务器内部错误

## 鉴权流程

1. 客户端调用注册或登录接口。
2. 服务端校验成功后签发 JWT。
3. 客户端访问受保护接口时，在请求头中携带 `Authorization: Bearer <jwt-token>`。
4. 鉴权中间件校验 token 签名和过期时间。
5. 中间件根据 token 中的用户 ID 查询数据库。
6. 用户存在且状态正常时，请求继续进入 controller；否则返回 `401`。

## 当前实现文件

- `controllers/auth_controller.go`：注册、登录、当前用户接口
- `controllers/user_controller.go`：用户列表接口
- `middlewares/auth.go`：JWT 签发、校验和 Gin 鉴权中间件
- `models/user_model.go`：用户表映射、注册、登录、查询逻辑
- `responses/response.go`：统一响应结构
- `routers/router.go`：HTTP 路由注册
- `docs/openapi.json`：当前已实现接口的 OpenAPI 描述

## 验证示例

注册：

```bash
curl -X POST http://127.0.0.1:8080/api/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","password":"123456","nickname":"Alice"}'
```

登录：

```bash
curl -X POST http://127.0.0.1:8080/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"alice","password":"123456"}'
```

获取当前用户：

```bash
curl http://127.0.0.1:8080/api/me \
  -H 'Authorization: Bearer <jwt-token>'
```

获取用户列表：

```bash
curl http://127.0.0.1:8080/admin/users \
  -H 'Authorization: Bearer <jwt-token>'
```
