# 版本六：群管理与消息状态版

## 版本目标

在基础 IM 主链路完成后，补齐群聊管理和消息状态能力，让会话从“能聊”变成“可管理”。

v6 不新增核心表，主要复用 `conversations`、`conversation_members` 和 `messages`。

## 前置基础

- v2 已实现会话和成员关系
- v3 已实现消息和消息状态字段
- v4 已实现在线消息投递

## 数据库文档

本版本不新增数据库表，依赖：

- [数据库表创建-会话](../数据库/数据库表创建-会话.md)
- [数据库表创建-消息](../数据库/数据库表创建-消息.md)
- [数据库表关系](../数据库/数据库表关系.md)

## 本版本新增能力

- 群聊添加成员
- 群聊移除成员
- 群成员退出群聊
- 群主设置或取消管理员
- 修改群聊资料
- 撤回消息
- 删除消息

## 权限矩阵

| 操作 | owner | admin | member |
|---|---:|---:|---:|
| 添加成员 | 是 | 是 | 否 |
| 移除 member | 是 | 是 | 否 |
| 移除 admin | 是 | 否 | 否 |
| 移除 owner | 否 | 否 | 否 |
| 设置 admin/member | 是 | 否 | 否 |
| 修改群资料 | 是 | 是 | 否 |
| 撤回自己的消息 | 是 | 是 | 是 |
| 撤回他人消息 | 是 | 是 | 否 |
| 删除自己的消息 | 是 | 是 | 是 |
| 删除他人消息 | 是 | 是 | 否 |

## 接口设计

### 添加群成员

```text
POST /api/conversations/{conversation_id}/members
```

请求体：

```json
{
  "user_ids": [2, 3]
}
```

规则：

- 只支持群聊
- 群主和管理员可以添加成员
- 已经在群里的 active 成员忽略
- `left` 或 `removed` 成员重新加入时，可以更新状态为 `active`

### 移除群成员

```text
DELETE /api/conversations/{conversation_id}/members/{user_id}
```

规则：

- 只支持群聊
- 群主可以移除管理员和普通成员
- 管理员只能移除普通成员
- 不能移除群主
- 被移除成员 `status` 更新为 `removed`

### 退出群聊

```text
POST /api/conversations/{conversation_id}/leave
```

规则：

- 只支持群聊
- 普通成员和管理员可以主动退出
- 群主不能直接退出；需要先转让群主或解散群聊
- 退出后成员 `status` 更新为 `left`

### 设置成员角色

```text
PATCH /api/conversations/{conversation_id}/members/{user_id}/role
```

请求体：

```json
{
  "role": "admin"
}
```

规则：

- 只支持群聊
- 只有群主可以设置角色
- 可设置角色为 `admin` 或 `member`
- 不能通过该接口修改群主

### 修改群聊资料

```text
PATCH /api/conversations/{conversation_id}
```

请求体：

```json
{
  "title": "新的群名称",
  "avatar_url": "https://example.com/group.png"
}
```

规则：

- 只支持群聊
- 群主和管理员可以修改
- `title` 不能为空

### 撤回消息

```text
PATCH /api/messages/{message_id}/recall
```

规则：

- 消息发送者可以撤回自己的消息
- 群主和管理员可以撤回群内消息
- 撤回后 `messages.status = recalled`
- 撤回消息仍可在历史记录中返回，但内容可以置空
- 撤回事件可以通过 WebSocket 投递给在线成员

### 删除消息

```text
PATCH /api/messages/{message_id}/delete
```

规则：

- v6 先实现对所有人删除
- 消息发送者、群主、管理员可以删除
- 删除后 `messages.status = deleted`
- 历史消息接口默认不返回 `deleted` 消息

## 核心流程

### 添加成员

1. 校验当前用户是群聊 active 成员。
2. 校验当前用户角色是 `owner` 或 `admin`。
3. 校验待添加用户存在。
4. 查询已有成员记录。
5. 已存在但非 active 的成员更新为 `active`。
6. 不存在的成员插入新记录。
7. 可生成系统消息记录本次成员变化。

### 移除成员

1. 校验当前用户是群聊 active 成员。
2. 校验当前用户有权限移除目标成员。
3. 校验目标成员不是群主。
4. 更新目标成员 `status = removed`。
5. 可生成系统消息记录本次成员变化。

### 撤回消息

1. 查询消息和所属会话。
2. 校验当前用户是会话 active 成员。
3. 判断当前用户是否是发送者、群主或管理员。
4. 更新 `messages.status = recalled`。
5. 通过 WebSocket 投递消息状态变更事件。

## 边界规则

- 单聊不支持添加成员、移除成员、退出
- 群主不能被移除
- 管理员不能修改群主角色
- 被移除或退出的成员不能继续发送消息
- 被移除或退出的成员是否能看历史消息，由接口规则决定；建议只能看到退出前消息
- v6 不做群解散，避免破坏已有会话数据

## 错误处理

- `400`：参数校验失败
- `401`：未登录或登录已失效
- `403`：权限不足
- `404`：会话、成员或消息不存在
- `409`：状态冲突，例如移除群主
- `500`：服务器内部错误

## 验证方式

1. 群主添加新成员，确认新成员能看到群聊。
2. 管理员移除普通成员，确认普通成员不能发送消息。
3. 普通成员退出群聊，确认会话列表不再显示该群。
4. 群主设置管理员，确认管理员可以添加成员。
5. 消息发送者撤回消息，确认历史消息状态为 `recalled`。
6. 管理员删除群内消息，确认历史消息默认不返回该消息。

## 本版本不包含

- 群解散
- 群主转让
- 针对个人的消息删除
- 消息编辑
- 禁言完整能力
