# 版本二：会话基础版

## 版本目标

在 v1 用户体系和 JWT 鉴权基础上，落地单聊、群聊和会话成员关系。

v2 只负责“谁和谁在一个会话里”，不实现消息发送、历史消息、WebSocket 实时投递和未读数。

## 前置基础

- v1 已实现用户注册、登录和 JWT 鉴权
- 所有 v2 接口都需要 `Authorization: Bearer <jwt-token>`
- 用户身份统一从鉴权中间件写入的当前用户读取

## 数据库文档

本版本涉及数据库表：

- [数据库表创建-会话](../数据库/数据库表创建-会话.md)
- [数据库表关系](../数据库/数据库表关系.md)

## 本版本新增能力

- 创建或获取单聊会话
- 创建群聊会话
- 查询当前用户会话列表
- 查询单个会话详情
- 维护会话成员关系和成员状态

## 数据库变更

新增表：

- `conversations`
- `conversation_members`

v2 不写入消息，所以 `conversations.last_message_id` 和 `conversations.last_message_at` 可以为空。

## 代码落地清单

建议按当前项目 MVC 风格新增和修改文件。

新增文件：

- `models/conversation_model.go`：定义 `Conversation`、`ConversationMember`、DTO 和会话查询/创建方法
- `controllers/conversation_controller.go`：处理会话 HTTP 入参、鉴权用户读取和响应

修改文件：

- `routers/router.go`：注册 `/api/conversations` 路由，并统一挂 `middlewares.AuthRequired(cfg.JWT)`
- `docs/openapi.json`：补充 v2 新增接口描述

不需要新增 service 层。当前项目业务逻辑主要放在 `models`，v2 继续沿用这个方式，避免一次性改动项目结构。

## Go 模型设计

`models/conversation_model.go` 建议包含以下结构：

```go
const (
	ConversationTypeDirect = "direct"
	ConversationTypeGroup  = "group"

	MemberRoleOwner  = "owner"
	MemberRoleAdmin  = "admin"
	MemberRoleMember = "member"

	MemberStatusActive  = "active"
	MemberStatusLeft    = "left"
	MemberStatusRemoved = "removed"
)

type Conversation struct {
	ID            int64      `gorm:"column:id;primaryKey;autoIncrement" json:"id"`
	Type          string     `gorm:"column:type" json:"type"`
	DirectKey     *string    `gorm:"column:direct_key" json:"direct_key,omitempty"`
	Title         *string    `gorm:"column:title" json:"title,omitempty"`
	AvatarURL     *string    `gorm:"column:avatar_url" json:"avatar_url,omitempty"`
	OwnerID       *int64     `gorm:"column:owner_id" json:"owner_id,omitempty"`
	LastMessageID *int64     `gorm:"column:last_message_id" json:"last_message_id,omitempty"`
	LastMessageAt *time.Time `gorm:"column:last_message_at" json:"last_message_at,omitempty"`
}

type ConversationMember struct {
	ID                int64      `gorm:"column:id;primaryKey;autoIncrement" json:"id"`
	ConversationID    int64      `gorm:"column:conversation_id" json:"conversation_id"`
	UserID            int64      `gorm:"column:user_id" json:"user_id"`
	Role              string     `gorm:"column:role" json:"role"`
	JoinedAt          time.Time  `gorm:"column:joined_at" json:"joined_at"`
	MutedUntil        *time.Time `gorm:"column:muted_until" json:"muted_until,omitempty"`
	LastReadMessageID *int64     `gorm:"column:last_read_message_id" json:"last_read_message_id,omitempty"`
	LastReadAt        *time.Time `gorm:"column:last_read_at" json:"last_read_at,omitempty"`
	Status            string     `gorm:"column:status" json:"status"`
}
```

需要实现的 model 方法：

- `func (Conversation) TableName() string`
- `func (ConversationMember) TableName() string`
- `func BuildDirectKey(userID int64, targetUserID int64) (string, error)`
- `func GetOrCreateDirectConversation(userID int64, targetUserID int64) (ConversationDTO, error)`
- `func CreateGroupConversation(ownerID int64, title string, avatarURL string, memberIDs []int64) (ConversationDTO, error)`
- `func ListConversations(userID int64) ([]ConversationListItemDTO, error)`
- `func GetConversationDetail(userID int64, conversationID int64) (ConversationDetailDTO, error)`
- `func EnsureActiveMember(userID int64, conversationID int64) (ConversationMember, error)`

新增领域错误：

- `ErrConversationNotFound`
- `ErrConversationForbidden`
- `ErrInvalidConversationType`
- `ErrInvalidMember`
- `ErrCannotCreateDirectWithSelf`

这些错误由 controller 转换为稳定 HTTP 状态码。

`controllers/conversation_controller.go` 建议包含：

```go
type ConversationController struct{}

type CreateDirectConversationRequest struct {

type CreateGroupConversationRequest struct {
	Title     string  `json:"title" binding:"required"`
	AvatarURL string  `json:"avatar_url"`
	MemberIDs []int64 `json:"member_ids"`

```

需要实现的方法：

- `func (ctl ConversationController) Index(c *gin.Context)`
- `func (ctl ConversationController) Show(c *gin.Context)`
- `func (ctl ConversationController) CreateDirect(c *gin.Context)`
- `func (ctl ConversationController) CreateGroup(c *gin.Context)`

controller 只做：

- 读取当前用户：`middlewares.CurrentUser(c)`
- 绑定 JSON 或路径参数
- 调用 `models` 方法
- 转换错误到 HTTP 响应
- 使用 `responses.Success` 和 `responses.Error` 输出统一结构

## 路由落地

`routers/router.go` 在 `/api` 下新增鉴权分组：

```go
conversations := api.Group("/conversations", middlewares.AuthRequired(cfg.JWT))
{
	conversations.GET("", conversationController.Index)
	conversations.GET("/:conversation_id", conversationController.Show)
	conversations.POST("/direct", conversationController.CreateDirect)
	conversations.POST("/groups", conversationController.CreateGroup)
}
```

## 实现顺序

1. 在 MySQL 中执行 [数据库表创建-会话](../数据库/数据库表创建-会话.md) 的建表 SQL。

4. 实现 `GetOrCreateDirectConversation`，使用事务写入单聊和两个成员。
5. 实现 `CreateGroupConversation`，使用事务写入群聊和成员。
6. 实现 `ListConversations` 和 `GetConversationDetail`。
7. 新增 `controllers/conversation_controller.go`。
8. 修改 `routers/router.go` 注册路由。
9. 更新 `docs/openapi.json`。
10. 使用 curl 完成端到端验证。

## 接口设计

### 获取会话列表

```text
GET /api/conversations
```

返回当前用户参与且成员状态为 `active` 的会话。

响应示例：

```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "type": "direct",
      "title": "",
      "avatar_url": "",
      "owner_id": 0,
      "member_count": 2,
      "last_message_id": 0,
      "last_message_at": ""
    }
  ],
  "message": "获取会话列表成功"
}
```

### 查询会话详情

```text
GET /api/conversations/{conversation_id}
```

只允许会话成员访问。

响应示例：

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "type": "group",
    "title": "测试群",
    "avatar_url": "",
    "owner_id": 1,
    "members": [
      {
        "user_id": 1,
        "role": "owner",
        "status": "active"
      },
      {
        "user_id": 2,
        "role": "member",
        "status": "active"
      }
    ]
  },
  "message": "获取会话详情成功"
}
```

### 创建或获取单聊

```text
POST /api/conversations/direct
```

请求体：

```json
{
  "target_user_id": 2
}
```

成功响应：

```json
{
  "code": 200,
  "data": {
    "id": 1,
    "type": "direct",
    "direct_key": "1:2",
    "members": [
      {
        "user_id": 1,
        "role": "member"
      },
      {
        "user_id": 2,
        "role": "member"
      }
    ]
  },
  "message": "获取单聊会话成功"
}
```

规则：

- `target_user_id` 必须存在
- 不允许和自己创建单聊
- 单聊 `direct_key` 格式为 `min_user_id:max_user_id`
- 如果 `direct_key` 已存在，直接返回已有会话
- 如果不存在，在同一事务中创建 `conversations` 和两个 `conversation_members`

### 创建群聊

```text
POST /api/conversations/groups
```

请求体：

```json
{
  "title": "测试群",
  "avatar_url": "",
  "member_ids": [2, 3]
}
```

成功响应：

```json
{
  "code": 200,
  "data": {
    "id": 2,
    "type": "group",
    "title": "测试群",
    "owner_id": 1,
    "member_count": 3
  },
  "message": "创建群聊成功"
}
```

规则：

- `title` 必填
- 当前用户自动成为群主
- `member_ids` 不需要包含当前用户，服务端会自动补齐
- `member_ids` 中的用户必须存在
- 重复成员 ID 需要去重
- 创建群聊必须在同一事务中写入 `conversations` 和 `conversation_members`

## 核心流程

### 创建单聊

1. 从 JWT 中获取当前用户 ID。
2. 校验 `target_user_id` 存在且不等于当前用户。
3. 对两个用户 ID 排序。
4. 生成 `direct_key`。
5. 查询 `conversations.direct_key` 是否已存在。
6. 已存在则返回已有会话。
7. 不存在则开启事务。
8. 写入 `conversations`，`type = direct`。
9. 写入两个 `conversation_members`，角色均为 `member`。
10. 提交事务并返回会话。

### 创建群聊

1. 从 JWT 中获取当前用户 ID。
2. 校验群名称。
3. 合并当前用户 ID 和 `member_ids`，并去重。
4. 校验所有用户 ID 都存在。
5. 开启事务。
6. 写入 `conversations`，`type = group`，`owner_id = 当前用户 ID`。
7. 当前用户写入 `conversation_members.role = owner`。
8. 其他用户写入 `conversation_members.role = member`。
9. 提交事务并返回群聊。

### 获取会话列表

1. 根据当前用户 ID 查询 `conversation_members`。
2. 只保留 `status = active` 的成员记录。
3. 关联 `conversations` 获取会话基础信息。
4. 统计每个会话的 active 成员数量。
5. 按 `last_message_at` 倒序；为空时按 `updated_at` 倒序。

## 边界规则

- 非会话成员不能查看会话详情
- 单聊不能重复创建
- 群聊可以创建多个同名群
- v2 不计算未读数
- v2 不返回最后一条消息详情
- v2 不支持退出群聊、移除成员、修改群名称

## 错误处理

- `400`：参数校验失败
- `401`：未登录或登录已失效
- `403`：无权访问该会话
- `404`：用户或会话不存在
- `409`：成员关系冲突
- `500`：服务器内部错误

## 验证方式

1. 注册并登录两个用户。
2. 使用用户 A token 调用 `POST /api/conversations/direct` 创建和用户 B 的单聊。
3. 重复调用同一个接口，确认返回同一个会话 ID。
4. 使用用户 A token 创建群聊。
5. 使用群成员 token 调用 `GET /api/conversations`，确认能看到群聊。
6. 使用非成员 token 查询会话详情，确认返回 `403`。

## 本版本不包含

- 消息发送
- 历史消息
- 未读数
- WebSocket
- 设备上报
- 离线推送
